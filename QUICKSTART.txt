Problem: "ModuleNotFoundError: No module named 'requests'"
Solution:
  pip install requests

Problem: "Port 5000 already in use"
Solution:
  # Find process using port 5000 and kill it
  # OR run Flask on different port:
  flask run --port 5001

Problem: Browser shows "Connection refused"
Solution:
  - Make sure Flask server is still running in terminal
  - Check it says "Running on http://127.0.0.1:5000"
  - Try hard refresh (Ctrl+F5) in browser

Problem: Genres showing "Unknown Genre" for popular songs
Solution:
  - Check internet connection (MusicBrainz needs access)
  - Check MusicBrainz is reachable:
    curl https://musicbrainz.org/ws/2/ping
  - Try different song title
  - Check Flask logs for MusicBrainz error

Problem: Very slow response (>10 seconds)
Solution:
  - This is expected for first search (1.1sec per song)
  - Subsequent searches should be <200ms
  - Check cache stats - if all hits, it's from cache
  - If all misses, MusicBrainz is slow today

WHAT'S ACTUALLY HAPPENING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

When you search "Blinding Lights The Weeknd":

1. Browser sends: GET /api/search?query=Blinding%20Lights%20The%20Weeknd

2. Flask server:
   a. Calls YouTube Music API → Gets raw song data
   b. Calls EnrichmentService.enrich_songs()
   c. For each song:
      - Searches MusicBrainz for recording
      - Extracts genres, release year
      - Infers language from title/artist
   d. Returns fully enriched JSON

3. Browser receives: 
   {
     "success": true,
     "results": [
       {
         "title": "Blinding Lights",
         "artists": ["The Weeknd"],
         "genres": ["Pop", "Electronic", "Dance-pop"],
         "release_year": 2019,
         "language": "English",
         ...
       }
     ]
   }

4. Frontend displays in table

No client-side enrichment. Everything handled server-side!
